# Should the game be FPS or something else?

Not FPS.

For your idea, the best experience is:

2D shop tycoon / idle business simulator

The ideal style is a mix of:

Idle clicker + shop management + light tycoon + upgrade simulator

Think of the player experience like this:

Customer enters → buys item → money pops up → player restocks → buys upgrade → shop looks better → more customers arrive → income grows faster

That loop is perfect for the feeling you want: “I am making money. I am growing. I am getting richer.”

Best visual format

Use one of these:

Top-down 2D shop view
Best for HTML/CSS/JS. Easy to build, mobile-friendly, easy to use with free assets.
Isometric 2.5D shop view
More beautiful and more “tycoon-like,” but harder to build.
Dashboard + animated shop scene
Easiest MVP. The shop is shown visually, but most interaction happens through buttons, cards, and upgrades.

My recommendation:

Start with top-down 2D or dashboard + animated shop scene.

###############################################################

# Should you have saves?

Yes, but do not use a backend at first.

You do not need Django, DRF, PostgreSQL, accounts, or paid hosting for the first version.

Use browser storage:

localStorage for simple saves

So the best MVP approach is:

No account
No login
No database
Save progress in the browser
Add “Reset Shop” button
Add “Export Save” / “Import Save” later

This solves the refresh problem without hosting costs.

Open-source / free asset sources

Use mostly CC0/public-domain assets when possible. CC0 is the safest because it is basically “no rights reserved” to the fullest extent allowed.

Good sources:

Kenney
Best first choice. Kenney says its game assets are CC0/public domain and can be used in commercial projects; attribution is not required.
OpenGameArt
Good, but check each asset license carefully. OpenGameArt includes CC0, CC-BY, CC-BY-SA, OGA-BY, public domain, and other licenses.
itch.io free game assets
Good for sprites, UI kits, tilesets, sounds, and backgrounds. itch.io also has filters for CC0 game assets.

Keep a simple spreadsheet like:

Asset name
Creator
Source
License
Attribution required?
Used in which game?

This protects you later.
